# jq

Useful jq patterns I picked up:

## Core principles

- Write the test first when fixing a bug - prevents regressions.
- Convention over configuration reduces cognitive load.

## Applied to jq

With jq, the boundary validation principle is especially important because invalid inputs can cascade through the entire pipeline before failing with a cryptic error three layers deep.

## Anti-patterns to avoid

1. Don't cache jq results without a TTL
2. Don't share jq connections across threads without pooling
3. Don't log sensitive jq config values (seen this too many times)
